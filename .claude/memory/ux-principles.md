## Core philosophy

The app serves ~8 non-technical coaches. They will receive a URL, email, and password — and nothing else. No training, no documentation, no help page. The app must be completely self-explanatory.

**Zero onboarding is the goal, not a nice-to-have.** If a feature needs explanation, the UI isn't clear enough. Fix the UI, not the docs.

## Rules

- **No help pages** — a help page is an admission of UX failure. Fix the label, placeholder, or layout instead.
- **Contextual hints only** — use placeholder text, field hints, and empty states directly where confusion might occur.
- **Catalog is a scanning experience** — no clutter on cards. Comments, metadata, and detail belong on the detail page.
- **Simplicity is the goal, not a constraint** — always ask whether a feature can be simpler. The right amount of UI is the minimum a coach needs to succeed on their own.
- **Nav should feel personal** — "Hi, NAME ▾" dropdown over a generic Account link. Reduces cognitive load and adds warmth.

## How to apply

When proposing UI changes, ask: "Could a coach figure this out with zero explanation?" If no, simplify before building. When tempted to add a tooltip or instructions, fix the underlying UX problem instead.
